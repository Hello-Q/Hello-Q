    • 迁移相关的命令

      命令 用途
      makemigrations 将模型变更用Django的形式记录下来
      migrate 执行模型变更，同步到数据库
      sqlmigrate 显示每次迁移执行的sql语句
      showmigrations 显示某个应用的模型变更和迁移历史

    • 迁移文件详解

      字段 用途
      dependencies 本次迁移的依赖app和依赖迁移记录
      operations 本次迁移的相关操作

  • 懒加载与预加载

    • 懒加载

        django懒加载是指在存在关系的表中进行数据查询的时候，只查询本体数据，不查询关联对象的数据，调用关联对象数据时，会再次查询数据库。造成二次查询，对数据库造成压力。

    • 预加载

      • 方法

        方法 说明
        select_related 预加载单个关联对象
        prefetch_related 预加载多个关联对象

      • 例子

          user = User.objects.prefetch_related('menu')

        User为模型，menu为User模型中的外键，是一对多关系

        说明
        select_related 用于一对一、多对一
        prefetch_related 用于一对多、多对多

  • 数据库操作规范

    • 合理使用索引

      • 正确使索引
      • 在经常查询、排序、比较的字段建立索引

    • 使用迭代器itertor迭代较大的QuerySet

      • 查询数据较大时，使用迭代器节省内存

    • 对象属性缓存

      • 不可调用的属性会被ORM框架缓存
      • 可调用的属性不会被ORM框架缓存
        ```python user = User.objects.all()[0] user.name # 不可调用属性 user.menu.all() # 可调用属性 menu是外键 ``` > 通过函数方式调用的就是可调用属性，否则就是不可调用属性，不可调用属性不会被缓存，尽量储存在变量中以供下次使用

    • 数据库的操作由数据库来做

      • 过滤： 使用filter、exclude等属性
      • 聚合： 使用annotate函数进行聚合
      • 使用原生SQL查询

    • 使用检索时，尽量使用有索引的字段

    • 不进行不必要的检索

      • QuserSet使用values()、 value_list() 函数返回pyhton结构容器
      • 查询结果长度使用QuerySet.count()而不是len(QuerySet)
