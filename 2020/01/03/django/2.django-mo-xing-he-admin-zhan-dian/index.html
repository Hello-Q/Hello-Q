数据库配置

配置settings中的 DATABASES 变量

  • ENGINE 数据库引擎
    • django.db.backends.sqlite3
    • django.db.backends.postgresql
    • django.db.backends.mysql
    • django.db.backends.oracle
  • NAME 数据库的名称
    使用SQLite则指定文件的路径

mysql配置示例

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'HOST': '127.0.0.1',
        'PORT': 3306,
        'NAME': 'crm',
        'USER': 'root',
        'PASSWORD': 'crm2019',
    },
}

迁移数据库

Django 自带了一些app（INSTALLED_APPS列表中），使用这些app需要先把app所需要的表创建。

  • 生成迁移文件

    python manage.py makemigrations <app_name>

    忽略app_name 将为所用installed_apps列表中的app生成迁移文件

  • 执行迁移

    python manage.py migrate <app_name>

    忽略app_name 将为所用installed_apps列表中的app生成的迁移文件执行生效

  • 查看迁移sql语句

    python manage.py sqlmigrate <app_name> <index>

创建模型

Django的模型用类表示，继承django.db.models.Model类

示例

from django.db import models


class Question(models.Model):
    question_text = models.CharField(max_length=200)
    pub_date = models.DateTimeField('date published')


class Choice(models.Model):
    question = models.ForeignKey(Question, on_delete=models.CASCADE)
    choice_text = models.CharField(max_length=200)
    votes = models.IntegerField(default=0)
  • 每个字段都是Field类的实例，映射到数据库中对应一个字段

Django常用Field字段参考
字段|描述
-|-
AutoField|自增长字段
CharField|字符串类型
BoolField|布尔型
DateField|日期
DateTimeField|日期时间
EmailField|会检查是否合法Email地址
FileField|存文件的
ImageField|继承FileField，存图片的

字段参数

所有字段都有的参数

参数 描述
db_column=”” 数据库中的表名
primmary_key=True 设置数据库主键
verbose_name=”” 设置别名、备注
unique=True 设置唯一键
null=True 设置允许为空（数据库层级）
blank=True 设置允许为空（表达层面）
db_index 设置索引字段
help_text=”” 在表单中显示帮助信息
editable=Ture 设置允许用户编辑此字段

个别字段有的参数

  • CharField
    • max_length=100 设置允许的最大长度
  • Date、DateTime
    • unique_for_date=True 设置日期唯一
    • unique_for_month 设置月份唯一
    • auto_now=True 设置更新当前记录的时间
    • auto_now_add 设置新增当前记录的时间
  • Deccimal
    • max_digits=4 设置数据总位数为4位
    • decimal_places=2 设置小数位数为2位
  • 关系型字段的参数
    • relates_name 父表查询子表时使用
    • db_constraint bool值，是否启用物理外键
    • on_delete

      当一个被外键关联的对象被删除时，Django将模仿on_delete参数定义的SQL约束执行相应操作
      如下6种操作

    - CASCADE：模拟SQL语言中的ON DELETE CASCADE约束，将定义有外键的模型对象同时删除！（该操作为当前Django版本的默认操作！）
    - PROTECT:阻止上面的删除操作，但是弹出ProtectedError异常
    - SET_NULL：将外键字段设为null，只有当字段设置了null=True时，方可使用该值。
    - SET_DEFAULT:将外键字段设为默认值。只有当字段设置了default参数时，方可使用。
    - DO_NOTHING：什么也不做。
    - SET()：设置为一个传递给SET()的值或者一个回调函数的返回值。注意大小写

Django管理界面admin

创建管理员账号

执行

python manage.py createsuperuser

按照提示输入用户名、密码等信息

向admin注册app

在app的admin.py中写

from django.contrib import admin

from .models import Question

admin.site.register(Question)
