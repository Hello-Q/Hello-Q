面向对象与面向过程的区别

面向过程（Procedure Oriented）：以过程为核心，强调事件的流程、顺序

面向对象（Object Oriented）：以对象为核心，强调事件的角色、主体

案例

小明起床上班这件事；

从面向过程的角度看就是：1、起床；2、刷牙洗脸；3、开车上班。

从面向对象的角度看就是：主人公：小明；拥有的财产：床、牙刷、车；小明可以做什么：从床上站起来、刷牙、开车。

优缺点

面向过程

  • 优点：将问题流程化，简单化
  • 缺点：扩展性差（更新、维护、升级）

适合简单的一次性任务、脚本

面向对象

  • 优点：扩拓展性高
  • 缺点：编程复杂度相对高。

刚好与面向过程相反。

面向对象的实现 - 创建类

面向对象编程中，用类表示现实世界中的事物和情景

在Python中使用class关键字来创建类

class Dog():   # 1

        """初始化属性name和age"""
        self.name = name  # 4
        self.age = age

    def sit(self):  # 5
        """模拟小狗被命令蹲下"""

初始化方法属于魔术方法

魔术方法：

  • 魔术方法一般不需要自己手动调用，再某种情况下会自动调用执行

  • 魔术方法两侧会用两个下划线

封装是使用特殊语法，对成员属性和方法进行包装，达到保护和隐藏的目的，对成员属性进行访问限制

2. 受保护的protected

对于Person类中的age属性，他不应该被更改，而是通过方法每过一年增加一岁，所以他应该设计成受保护的属性。

class Person():
    ...
    _age = None

    def __init__(self, age):
        self._age = age

xm = Person('小明', 18, 85)
print(xm._age)
>>> 18
xm._age = 19
>>> 19

通过在属性前面增加一个下划线，可以将属性设置成受保护的属性。但实际上在python中并没有对受保护的属性做任何处理，这样的变量仍然可以在外部访问、修改。但是，按照约定俗成的规定，当看到这样的变量时，意思就是，“虽然我可以被访问，但是，请把我视为受保护变量，不要随意访问”。

3. 私有的private

对于Person类中的score属性，分数不想让别人知道，所以应该设置成私有变量。

class Person():

    def __init__(self, score):
        self.__score = score    

    def get_score(self):
        print(self.__score)

    def set_score(self, score):
        if score >= 0:
            self.__score = score

通过在属性前面增加两个下划线，可以将属性设置成私有属性。外部无法直接访问，但是可以通过添加_Persoon的方式进行直接访问。但强烈不推荐这么做，因为私有变量不应该再外部访问，切不同的版本，Python解释权会将名称翻译成不同的名字。

类可以给私有变量设置修改的方法和获取的方法，在类的外部可以通过相应的方法对私有属性进行修改和访问，并且这样做的好处是，还可以对用户传入的值进行校验。

方法的封装

对于方法，方式是相同的，也是通过添加下滑线的方式进行对方法的封装。

继承

在OOP程序设计中，定义一个class的时候，可以从某个现有的class继承，新的class称为子类（Subclass）货派生类，而被继承的class称为基类、父类或超类（Base class、Super class）。

继承后的子类自动获得父类除私有属性和方法外的所有属性和方法。

继承的语法

class Animal(object):
    age = None

    def run(self):
        print('Animal is running...')

class Dog(Animal):
    pass 

class Cat(Animal):
    pass

dog = Dog()
cat = Cat()

dog.run()
cat.run()

输出：

Animal is running...
Animal is running...

Dog类继承了Animal。对于Dog来说，Animal就是它的父类，对于Animal来说，Dog就是它的子类。Cat和Dog类似。

并且Dog和Cat自动拥有了Animal的属性age和方法。

子类方法、属性重写

Dog与Cat运行run是的结果应该是不同的。可以在子类中对父类的方法进行重写。

class Dog(Animal):

    def run(self):
        print('Dog is running...')

class Cat(Animal):

    def run(self):
        print('Cat is running...')

dog = Dog()
cat = Cat()

dog.run()
cat.run()

输出

Dog is running...
Cat is running...

当子类拥有和父类相同名称的方法或属性时，会对父类的方法重写。实例化的对象将按照重写后的方法运行。

继承特点：

  • 子类继承父类后，自动拥有父类除私有属性、方法之外的所有方法。
  • 子类可以重新父类方法、属性
  • 子类可以拓展父类方法、属性（新增属性、方法）
  • 子类可以通过super().父类方法名的方式调用父类方法。

单继承与多继承

  • 单继承：一个类只继承一个父类
  • 多继承：一个类继承多个父类
菱形继承（钻石继承）
graph TB
A --> B
A --> C
B --> D
C --> D

B、C类同时继承A类（单继承），D类继承B和C类（多继承）此种方式成为菱形继承（钻石继承）

菱形继承中的super()调用顺序：

再使用super()调用父类方法时，实际是按照MRO列表的顺序查找上级类调用的。MRO列表可以通过class.__mro__来查看继承关系。

class A():
    pass
class B(A):
    pass
class C(A):
    pass
class D(B, C):
    pass

print(D.__mro__)

输出

(<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

使用super()调用父类方法时，会把当前的self对象带入父类。

继承关系的检测

使用issubclass(B, A)可以检测B类是否是A的子类，返回bool值

多态

不带继承关系的多态

对于同一个方法，调用对象的不同会有不同的结果

class Duck():
    def fly(self):
        print('鸭子开始贴地飞行了')

class Swan():
    def fly(self):
        print('天鹅翱翔空中')

class Plane():
    def fly(self):
        print('飞机飞在云彩上')

def fly(obj):
    obj.fly()

duck = Duck()
fly(duck)

swan = Swan()
fly(swan)

plank = Plane()
fly(plank)

输出：

鸭子开始贴地飞行了
天鹅翱翔空中
飞机飞在云彩上

由于向fly()函数中传入的参数类型（不同的类的对象，类型不同）不同，执行函数后输出的结果也就不同，这就是多态的。

带有继承关系的多态（应用更广）

class USB():
    def start():
        pass

class Keyboard(USB):
    def start(self):
        print('键盘已插入，我可以打字')

class Mouse(USB):
    def start(self):
        print('鼠标已插入，我可以操作鼠标')

class Udisk(USB):
    def start(self):
        print('U盘已插入，我可以储存数据')


k = Keyboard()
m = Mouse()
u = Udisk()

k.start()
m.start()
u.start()

输出

键盘已插入，我可以打字
鼠标已插入，我可以操作鼠标
U盘已插入，我可以储存数据

定义一个基础的类USB，其他的子类重写父类的方法，调用同一个方法，由于调用的对象不同，执行的结果也不相同。

多态的作用

让具有不同功能的函数可以使用相同的函数名，这样就可以用一个函数名调用不同内容(功能)的函数。

多态的特点

1、只关心对象的实例方法是否同名，不关心对象所属的类型；
2、对象所属的类之间，继承关系可有可无；
3、多态的好处可以增加代码的外部调用灵活度，让代码更加通用，兼容性比较强；
4、多态是调用方法的技巧，不会影响到类的内部设计。

实例属性和类属性

实例属性属于实例，类属性属于类。当实例属性与类属性同名时，会覆盖类属性。

给实例绑定属性的方法是通过实例变量，或者通过self变量。

class Animal():

print('类属性还存在', Dog.name)

输出

修改前的小狗2名字 xiaobai
修改后的小狗1名字 xiaohei
类属性还存在 xiaobai
修改后小狗2的名字 xiaobai

类属性是共用的，当一个实例的属性与类属性名称相同时，会将自己实例的属性覆盖但不会影响类属性和其他实例。

魔术方法

方法相关的函数：

方法 触发时机 作用
__init__ 初始化方法 对象实例化时 完成实例初始化，一般成员赋值，打开资源等。
__del__ 析构方法 对象被销毁时 关闭一些资源等
__len__ 当使用len()去检测对象时 代替len检测对象中的某个信息，一定有一个整数返回值

__new__ 构造方法

触发时机：对象实例化之前(在__init__之前)

作用： 管理对象创建过程

参数：cls接受当前类。和其他参数，其他参数根据初始化方法

返回值：一般返回object.__new__(cls)进行新对象的创建，返回None将不创建对象。

注意：构造方法的参数除第一个外，其他要保持一致。

class Person(object):

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __new__(cls, name, age):
        if 0 < age < 150:
            return object.__new__(cls)
            # return super(Person, cls).__new__(cls)
        else:
            return None

print(Person('Tom', 10))
print(Person('Mike', 200))

输出

Person({'name': 'Tom', 'age': 10})
None

对对象创建过程控制，如果年龄不在0到150之间，将不创建对象。（没人年龄是负数，年龄能大于150，至少现在没有）

__call__

触发时机：此方法会在对象被当做函数使用时调用

作用：可以归纳类中的一些方法，一起调用

class Demo():

    def test_a(self):
        print("test_a被调用")

    def test_b(self):
        print("test_b被调用")        

    def __call__(self):
        print('对象被当做函数调用')
        self.test_a()
        self.test_b()

demo = Demo()
demo()

输出

对象被当做函数调用
test_a被调用
test_b被调用

成员相关魔术方法：

__getattribute__ 访问属性触发

触发时机：调用对象属性时调用

返回值：返回处理后的成员值

参数：self，item（调用的参数）

注意：在方法内不能再试用self去访问属性，否则会触发递归，应该使用object.__getattribute__访问

class Demo():
    name = '张晓明'
    def __getattribute__(self, item):
        res = object.__getattribute__(self, item)
        return res[0] + '*' + res[-1]

obj = Demo()
print(obj.name)

输出

张*明
__getattr__ 访问不存在属性触发

触发时机：访问不存在的成员时触发

作用：防止访问不错在的成员时报错

参数：self接受道歉对象，item接受访问的成员名称

返回值：返回处理后的成员值

class Demo():
    name = '张晓明'
    def __getattr__(self, item):
        item = 10
        return item


obj = Demo()
print(obj.age)

输出

10

当前程序访问了一个不错在的属性age，__getattr__将这个属性赋值为10并且返回。

__setattr__ 属性修改触发

触发时机：修改或添加属性时触发

作用：可以限制管理对象成员的修改

参数：1self接受对象，2设置的成员名，3设置的成员值

返回值：None

注意事项：不能在这个函数内设置成员属性，否则触发递归。需要借助object.__setattr__(self, attr, value)

class Demo():
    name = '张晓明'

    def __setattr__(self, key, value):
        print('你想把', key, '修改为', value)
        value = '小红'
        object.__setattr__(self, key, value)
        print('但是我把他修改成了', value)

obj = Demo()
obj.name = '小洪'
print(obj.name)

输出

你想把 name 修改为 小洪
但是我把他修改成了 小红
小红
__delattr__ 删除成员时触发

作用：限制删除成员的操作

参数：1self接受当前对象， 2item当前删除的对象名称

返回值：无

注意：当前方法中不能直接删除成员，可以借助object.__delattr__(slef, item)

class Demo():

    def __delattr__(self, item):
        object.__delattr__(self, 'age')
        print('你想删除', item, '但是我TM把age删除了')

obj = Demo()
obj.name = '张晓明'
obj.age = 19
del obj.name

输出

你想删除 name 但是我TM把age删除了

类的成员描述符

概念

当一个类包含三个魔术方法（__get__,__set__,__delete__)任意一到三个时，这个类称为描述符类。

作用

对一个类的某个成员进行一个详细的管理操作（获取、赋值、删除）

描述符是代理一个类中成员的操作，描述符属于类，只能定义为类的属性。

成员的访问顺序

graph TD
__getattribute__ --> 数据描述符 --> 对象成员 --> 类成员 --> 非数据描述符 --> 父类成员 --> __getattr__

数据描述符：至少实现了__get__和__set__的描述符类

非数据描述符：只实现了除__set__外的方法的描述符类

案例

定义一个类，记录一个学生的姓名和成绩，，成绩在0-100之间。

class Score:
    score = None

    def __get__(self, instance, owner):
        print('获取成员属性')
        return self.score

    def __set__(self, instance, value):
        print('设置成员属性')
        if 0 <= value <= 100:
            self.score = value
        else:
            raise ValueError(value, '不在合法分数区间')


class Student:
    name = None
    score = Score()

    def __init__(self, name, score):
        self.name = name
        if 0 <= score <= 100:
            self.score = score
        else:
            raise ValueError(score, '不在合法分数区间')

    def print_info(self):
        print(f'学员信息:姓名：{self.name}，分数：{self.score}')


stu = Student('小明', 80)
print(stu.score)
stu.score = 89
print(stu.score)

输出

设置成员属性
获取成员属性
80
设置成员属性
获取成员属性
89

通过使用数据描述符，对修改属性的属性值进行判断，如果不符合标准，将引发ValueErro异常。

描述符的其他实现方式

使用property函数

class Student:
    def __init__(self, name, score):
        self.name = name
        self._score = self.score = score

    def get_score(self):
        print('get')
        return self._score

    def set_score(self, value):
        print('set')
        self._score = value

    def del_score(self):
        del self._score

    score = property(get_score, set_score, del_score)


stu = Student('小明', 89)
print(stu.score)
stu.score = 98
del stu.score

使用@property装饰器

class Student:
    _score = None
    @property
    def score(self):
        print('get')
        return self._score

    @score.setter
    def score(self, value):
        print('set')
        self._score = value

    @score.deleter
    def score(self):
        print('del')
        del self._score


stu = Student()
print(stu.score)
stu.score = 88
del stu.score

输出

get
None
set
del

一般使用描述类实现，因为不需要修改原来的类。

设计模式

设计模式（Design pattern）代表了最佳的实践，通常被有经验的面向对象的软件开发人员所采用。设计模式是软件开发人员在软件开发过程中面临的一般问题的解决方案。这些解决方案是众多软件开发人员经过相当长的一段时间的试验和错误总结出来的。

单例模式

在当前脚本中，一个类只能创建一个对象。这种情况称为单例。

class Demo:
    __ojb = None

    def __new__(cls, *args, **kwargs):
        if cls.__ojb:
            return cls.__ojb
        else:
            obj = object.__new__(cls)
            cls.__ojb = obj
            return obj

a = Demo()
b = Demo()
print(id(a) == id(b))

输出

True

多次创建返回的对象都是指向同一个地址（即只有一个对象）

Mixin混合设计模式

Mixin 的概念

Mixin通过多继承的方式实现。

Mixin是设计一个类，这个类只实现单一的功能，且该类只提供给其他类继承。

如果有多个功能，那么就需要设计多个Mixin类来实现各个功能。

Mixin类通常不会单独使用（实例化）而是混入到（继承）其他类中增加其他类的功能。

继承Mixin类的子类不应该依靠Mixin类，即子类及时不继承Mixin类也能正常使用，只是缺少了一个功能。

案例

狗和人都要睡觉、吃饭，就可以把睡觉和吃饭独立出来设计成一个Mixin类，然后供给需要吃饭和睡觉的类。

class EatMixin:
    def eat(self):
        print('正在吃东西。。。')


class SleepMixin:
    def sleep(self):
        print('正在睡大觉。。。')


class Person(EatMixin, SleepMixin):

    def study(self):
        print('正在学习。。。')

class Dog(EatMixin, SleepMixin):

    def look_house(self):
        print('正在看门护院。。。')

per = Person()
per.sleep()
per.study()
dog = Dog()
dog.sleep()
dog.look_house()

输出

正在睡大觉。。。
正在学习。。。
正在睡大觉。。。
正在看门护院

作用

  • 通过Mixin的设计模式，在不对原类修改的前提下，可以自由的添加新的功能。

  • 提高代码的复用性，所有有这个功能的类都可以通过继承的方式获得此功能。

原则

定义mixin类，应遵循以下原则：

  1. Mixin 实现的功能需要是通用的，并且是单一的，比如上例中两个 Mixin 类都适用于大部分子类，每个 Mixin 只实现一种功能，可按需继承。
  2. Mixin 只用于拓展子类的功能，不能影响子类的主要功能，子类也不能依赖 Mixin。比如上例中 Person 继承不同的 Mixin 只是增加了一些功能，并不影响自身的主要功能。如果是依赖关系，则是真正的基类，不应该用 Mixin 命名。
  3. Mixin 类自身不能进行实例化，仅用于被子类继承。
