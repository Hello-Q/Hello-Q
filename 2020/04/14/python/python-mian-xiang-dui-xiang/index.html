面向对象与面向过程的区别

面向过程（Procedure Oriented）：以过程为核心，强调事件的流程、顺序

面向对象（Object Oriented）：以对象为核心，强调事件的角色、主体

案例

小明起床上班这件事；

从面向过程的角度看就是：1、起床；2、刷牙洗脸；3、开车上班。

从面向对象的角度看就是：主人公：小明；拥有的财产：床、牙刷、车；小明可以做什么：从床上站起来、刷牙、开车。

优缺点

面向过程

  • 优点：将问题流程化，简单化
  • 缺点：扩展性差（更新、维护、升级）

适合简单的一次性任务、脚本

面向对象

  • 优点：扩拓展性高
  • 缺点：编程复杂度相对高。

刚好与面向过程相反。

面向对象的实现 - 创建类

面向对象编程中，用类表示现实世界中的事物和情景

在Python中使用class关键字来创建类

class Dog():   # 1
    """描述小狗的类"""  # 2

    def __init__(self, name, age):  # 3
        """初始化属性name和age"""
        self.name = name  # 4
        self.age = age

    def sit(self):  # 5
        """模拟小狗被命令蹲下"""
        print(self.name.title() + " is now sitting.")

    def roll_over(self):
        """模拟小狗打滚"""
        print(self.name.title() + " rolled_over!")

类的命名

类的命名建议使用驼峰命名法

类的组成

属性：一个变量，是类的描述

方法：一个函数，是类的功能

self形参

  • self在方法中是一个形参。
  • self在方法中代表的是当前自己这个对象。
  • 可以在类的内部代替对象本身调用方法、属性；添加方法、属性。

__init__初始化方法

初始化方法属于魔术方法

魔术方法：

  • 魔术方法一般不需要自己手动调用，再某种情况下会自动调用执行

  • 魔术方法两侧会用两个下划线

__init__方法会在对象实例化后自动调用执行。主要用于完类实例化后的对象的初始化。

__del__析构方法

与初始化方法相反，析构方法在对象被销毁时自动调用，主要可用于关闭一些资源（如关闭打开的文件）


class Log():

    def __init__(self):
        print("对象初始化")

    def __del__(self):
        print("对象被销毁")


log = Log()
>>> 对象初始化

del log
>>> 对象被销毁

log = 1
>>> 对象被销毁

对象被销毁的情况

  • Python解释器退出，所有对象都会被销毁
  • 使用del 删除一个对象
  • 当对象不在被引用时（变量指向了其他对象，当前对象即被销毁）

面向对象三大特性

封装

封装是使用特殊语法，对成员属性和方法进行包装，达到保护和隐藏的目的，对成员属性进行访问限制

封装的级别

1. 公有地public
# -*- coding: utf-8 -*-


class Person():
    name = None
    age = None
    score = None

    def __init__(self, name, age, sroce):
        self.name = name
        self.age = age
        self.sroce = sroce



xm = Person('小明', 18, 85)
print(xm.age)
>>> 18
xm.age = 19
>>> 19

对于公有的属性，不需要进行任何操作，共有的属性可以在类的内部使用，在类的外部，也可以使用（xm.age=19，直接对属性进行了修改）

obj.__dict__：可以查看对象的成员属性
class.__dict__：查看类的所有信息

2. 受保护的protected

对于Person类中的age属性，他不应该被更改，而是通过方法每过一年增加一岁，所以他应该设计成受保护的属性。

class Person():
    ...
    _age = None

    def __init__(self, age):
        self._age = age

xm = Person('小明', 18, 85)
print(xm._age)
>>> 18
xm._age = 19
>>> 19

通过在属性前面增加一个下划线，可以将属性设置成受保护的属性。但实际上在python中并没有对受保护的属性做任何处理，这样的变量仍然可以在外部访问、修改。但是，按照约定俗成的规定，当看到这样的变量时，意思就是，“虽然我可以被访问，但是，请把我视为受保护变量，不要随意访问”。

3. 私有的private

对于Person类中的score属性，分数不想让别人知道，所以应该设置成私有变量。

class Person():

    __score = None

    def __init__(self, score):
        self.__score = score    

    def get_score(self):
        print(self.__score)

    def set_score(self, score):
        if score >= 0:
            self.__score = score


print(xm.__sroce)  # 直接访问私有属性会报错

>>> {'name': 'xiaoming', '_age': 19, '_Preson__sroce': 95}

print(xm._Person__score)  # 可以访问私有属性
>>> 95

>>> 95

>>> 98

通过在属性前面增加两个下划线，可以将属性设置成私有属性。外部无法直接访问，但是可以通过添加_Persoon的方式进行直接访问。但强烈不推荐这么做，因为私有变量不应该再外部访问，切不同的版本，Python解释权会将名称翻译成不同的名字。

类可以给私有变量设置修改的方法和获取的方法，在类的外部可以通过相应的方法对私有属性进行修改和访问，并且这样做的好处是，还可以对用户传入的值进行校验。

方法的封装

对于方法，方式是相同的，也是通过添加下滑线的方式进行对方法的封装。

继承

在OOP程序设计中，定义一个class的时候，可以从某个现有的class继承，新的class称为子类（Subclass）货派生类，而被继承的class称为基类、父类或超类（Base class、Super class）。

继承后的子类自动获得父类除私有属性和方法外的所有属性和方法。

继承的语法

class Animal(object):
    age = None

    def run(self):
        print('Animal is running...')

class Dog(Animal):
    pass 

class Cat(Animal):
    pass

dog = Dog()
cat = Cat()

dog.run()
cat.run()

输出：

Animal is running...
Animal is running...

Dog类继承了Animal。对于Dog来说，Animal就是它的父类，对于Animal来说，Dog就是它的子类。Cat和Dog类似。

并且Dog和Cat自动拥有了Animal的属性age和方法。

子类方法、属性重写

Dog与Cat运行run是的结果应该是不同的。可以在子类中对父类的方法进行重写。

class Dog(Animal):

    def run(self):
        print('Dog is running...')

class Cat(Animal):

    def run(self):
        print('Cat is running...')

dog = Dog()
cat = Cat()

dog.run()
cat.run()

输出

Dog is running...
Cat is running...

当子类拥有和父类相同名称的方法或属性时，会对父类的方法重写。实例化的对象将按照重写后的方法运行。

继承特点：

  • 子类继承父类后，自动拥有父类除私有属性、方法之外的所有方法。
  • 子类可以重新父类方法、属性
  • 子类可以拓展父类方法、属性（新增属性、方法）
  • 子类可以通过super().父类方法名的方式调用父类方法。

单继承与多继承

  • 单继承：一个类只继承一个父类
  • 多继承：一个类继承多个父类
菱形继承（钻石继承）
graph TB
A --> B
A --> C
B --> D
C --> D

B、C类同时继承A类（单继承），D类继承B和C类（多继承）此种方式成为菱形继承（钻石继承）

菱形继承中的super()调用顺序：

再使用super()调用父类方法时，实际是按照MRO列表的顺序查找上级类调用的。MRO列表可以通过class.__mro__来查看继承关系。

class A():
    pass
class B(A):
    pass
class C(A):
    pass
class D(B, C):
    pass

print(D.__mro__)

输出

(<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

使用super()调用父类方法时，会把当前的self对象带入父类。

继承关系的检测

使用issubclass(B, A)可以检测B类是否是A的子类，返回bool值

多态

对于同一个方法，调用对象的不同会有不同的结果

class Duck():
    def fly(self):
        print('鸭子开始贴地飞行了')

class Swan():
    def fly(self):
        print('天鹅翱翔空中')

class Plane():
    def fly(self):
        print('飞机飞在云彩上')

def fly(obj):
    obj.fly()

duck = Duck()
fly(duck)

swan = Swan()
fly(swan)

plank = Plane()
fly(plank)

输出：

鸭子开始贴地飞行了
天鹅翱翔空中
飞机飞在云彩上

由于向fly()函数中传入的参数类型（不同的类的对象，类型不同）不同，执行函数后输出的结果也就不同，这就是多态的。

多态的作用

让具有不同功能的函数可以使用相同的函数名，这样就可以用一个函数名调用不同内容(功能)的函数。

多态的特点

1、只关心对象的实例方法是否同名，不关心对象所属的类型；
2、对象所属的类之间，继承关系可有可无；
3、多态的好处可以增加代码的外部调用灵活度，让代码更加通用，兼容性比较强；
4、多态是调用方法的技巧，不会影响到类的内部设计。
